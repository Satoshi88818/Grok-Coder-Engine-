Grok Coder v6 — Universal Verifiable Coding Engine

Built from v5 with all improvements implemented:

JSON-based coverage parsing across all languages (eliminates regex brittleness)

Multi-language mutation testing (stryker for JS/TS, cargo-mutants for Rust, gotests for Go)

Language-aware verifier prompts (verifier_prompt_addendum per handler)

Prompt cache instrumentation (hit/miss logging per iteration)

Hardened fallback sandbox (loud warning, opt-in flag, non-Python early-exit)

Stricter golden tests (min_coverage=78%, confidence assertions per component)

Hypothesis integration tightened (all languages get property-based tests where applicable)

""" Backend abstraction layer for Grok Coder v6. v6 change: prompt cache instrumentation. Each generate() call on OpenAICompatibleBackend inspects the usage object for prompt_cache_hit_tokens (vLLM) or x-cache header (TGI) and emits a structured log event. This lets us verify the cache is actually activating across iterations. """ from __future__ import annotations import logging from abc import ABC, abstractmethod from typing import Any, Dict, Iterator, List, Optional log = logging.getLogger(__name__) class BaseBackend(ABC): @abstractmethod def generate( self, messages: List[Dict[str, str]], *, max_tokens: int = 2048, temperature: float = 0.2, stream: bool = False, ) -> "str | Iterator[str]": ... @abstractmethod def health(self) -> bool: ... class HuggingFaceBackend(BaseBackend): """Local transformers pipeline backend.""" def __init__(self, model_name: str, load_in_4bit: bool = True, cpu_offload: bool = False): import torch from transformers import ( AutoModelForCausalLM, AutoTokenizer, BitsAndBytesConfig, pipeline, ) log.info("Loading HuggingFace model: %s (4bit=%s)", model_name, load_in_4bit) quantization_config = None if load_in_4bit: try: quantization_config = BitsAndBytesConfig(load_in_4bit=True) except Exception: log.warning("4-bit quantization unavailable; trying 8-bit fallback.") try: quantization_config = BitsAndBytesConfig(load_in_8bit=True) except Exception: log.warning("8-bit also unavailable; loading in full precision.") model = AutoModelForCausalLM.from_pretrained( model_name, quantization_config=quantization_config, device_map="auto" if not cpu_offload else "cpu", torch_dtype=torch.float16 if not cpu_offload else torch.float32, ) tokenizer = AutoTokenizer.from_pretrained(model_name) self._pipe = pipeline("text-generation", model=model, tokenizer=tokenizer) self._tokenizer = tokenizer def generate( self, messages: List[Dict[str, str]], *, max_tokens: int = 2048, temperature: float = 0.2, stream: bool = False, ) -> str: prompt = self._tokenizer.apply_chat_template( messages, tokenize=False, add_generation_prompt=True ) result = self._pipe( prompt, max_new_tokens=max_tokens, temperature=temperature, do_sample=temperature > 0, return_full_text=False, ) return result[0]["generated_text"] def health(self) -> bool: return self._pipe is not None class OpenAICompatibleBackend(BaseBackend): """ Generic backend for vLLM and TGI. v6: instruments prompt cache hits per iteration. vLLM exposes usage.prompt_cache_hit_tokens in the completion response object. We log this as a structured event so operators can verify cache efficiency across agentic iterations. """ def __init__( self, base_url: str, model: str, backend_type: str = "vllm", cache_instrumentation: bool = True, ): from openai import OpenAI self._client = OpenAI(base_url=base_url, api_key="local") self._model = model self._backend_type = backend_type self._cache_instrumentation = cache_instrumentation self._iteration_cache_stats: list[dict] = [] log.info("Using %s backend at %s (model=%s)", backend_type, base_url, model) def generate( self, messages: List[Dict[str, str]], *, max_tokens: int = 2048, temperature: float = 0.2, stream: bool = False, ) -> "str | Iterator[str]": if stream: return self._stream(messages, max_tokens=max_tokens, temperature=temperature) response = self._client.chat.completions.create( model=self._model, messages=messages, max_tokens=max_tokens, temperature=temperature, ) # v6: cache instrumentation if self._cache_instrumentation: self._log_cache_stats(response) return response.choices[0].message.content def _log_cache_stats(self, response: Any) -> None: """ Emit a structured cache hit/miss log event. vLLM adds prompt_cache_hit_tokens / prompt_cache_miss_tokens to usage. TGI may expose x-cache in response headers (not accessible via SDK). We log what we can observe from the usage object. """ usage = getattr(response, "usage", None) if usage is None: return hit_tokens = getattr(usage, "prompt_cache_hit_tokens", None) miss_tokens = getattr(usage, "prompt_cache_miss_tokens", None) prompt_tokens = getattr(usage, "prompt_tokens", None) stat = { "event": "prompt_cache_stat", "backend": self._backend_type, "prompt_tokens": prompt_tokens, "cache_hit_tokens": hit_tokens, "cache_miss_tokens": miss_tokens, "hit_rate": ( round(hit_tokens / prompt_tokens, 3) if hit_tokens is not None and prompt_tokens else None ), } self._iteration_cache_stats.append(stat) log.info("prompt_cache_stat %s", stat) def cache_summary(self) -> list[dict]: """Return all cache stats collected across iterations.""" return list(self._iteration_cache_stats) def _stream( self, messages: List[Dict[str, str]], max_tokens: int, temperature: float, ) -> Iterator[str]: with self._client.chat.completions.stream( model=self._model, messages=messages, max_tokens=max_tokens, temperature=temperature, ) as stream: for text in stream.text_stream: yield text def health(self) -> bool: try: models = self._client.models.list() return len(models.data) > 0 except Exception: return False def build_backend(config: Any) -> BaseBackend: if config.backend == "huggingface": return HuggingFaceBackend( model_name=config.model_name, load_in_4bit=config.load_in_4bit, cpu_offload=config.cpu_offload, ) if config.backend in ("vllm", "tgi"): if not config.backend_url: raise ValueError( f"GROK_BACKEND_URL must be set when using backend='{config.backend}'" ) return OpenAICompatibleBackend( base_url=config.backend_url, model=config.model_name, backend_type=config.backend, cache_instrumentation=getattr(config, "prompt_cache_instrumentation", True), ) raise ValueError(f"Unknown backend: {config.backend!r}")

# Grok Coder v6 — Universal Verifiable Coding Engine A local, multi-language, self-verifying agentic coding engine with objective confidence scoring, Docker sandboxing, JSON-based coverage parsing, multi-language mutation testing, language-aware verifier prompts, and prompt cache instrumentation. ## What's New in v6 | Feature | v5 | v6 | |---|---|---| | Coverage parsing | Regex per language (brittle) | JSON-first (robust), text fallback | | Mutation testing | Python only (mutmut) | Python + JS/TS (Stryker) + Rust (cargo-mutants) + Go (gremlins) | | Verifier prompts | Generic rubric | Language-aware addendum per handler | | Prompt cache | Enabled but unmonitored | Hit/miss instrumented per iteration | | Fallback sandbox | Silent opt-out | Explicit opt-in flag, loud warning always emitted | | Golden test gate | 60% coverage | 78% coverage + component assertions | | `overall_with_weights` | @property with argument (broken) | Callable method (fixed) | ## Architecture Axioms 1. **Correctness is measurable** — ConfidenceScore is computed from real signals. 2. **Security is language-agnostic** — tree-sitter scanner + seccomp sandbox. 3. **Extensibility is zero-boilerplate** — new language = one handler file + one Dockerfile. 4. **Structure over heuristics** — instructor + JSON coverage eliminates fragile parsing. 5. **Observability by default** — cache stats, mutation scores, and per-component confidence are all logged as structured events. ## Requirements - Python 3.11+ - Docker (required; fallback requires `--allow-fallback`) - CUDA GPU recommended; CPU fallback available - 4+ GB VRAM for 1.5B model; 8+ GB for 3B; 20+ GB for 7B ## Quick Start ```bash # 1. Known Limitations

restrictedpython fallback is not a true sandbox. Requires GROK_ALLOW_RESTRICTED_PYTHON_FALLBACK=true to activate.

Stryker (JS/TS mutation) is slow; disable with --mutation-langs python to restrict to Python only.

Go mutation testing via gremlins requires separate installation.

Rust sandbox image is ~500 MB due to the Rust toolchain + cargo-mutants.

vLLM structured output requires --guided-decoding-backend outlines in the vLLM server config.

Prompt cache hit rate logging is only available for vLLM backends (TGI doesn't expose token-level cache stats via the SDK).
